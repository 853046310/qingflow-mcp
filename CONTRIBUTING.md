# Contributing

Thanks for considering a contribution to `qingflow-mcp`.

## Development Setup

1. Install dependencies:

```bash
npm install
```

2. Build:

```bash
npm run build
```

3. Run locally:

```bash
npm run dev
```

## Pull Request Guidelines

1. Keep changes focused and small.
2. Update documentation when behavior changes.
3. Do not commit secrets or private tokens.
4. Ensure `npm run build` passes before opening a PR.

## Commit Style

Use clear commit messages, for example:

- `feat: add qf_operation_get tool`
- `fix: handle qingflow errorCode responses`
- `docs: update CLI install section`
